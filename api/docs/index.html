Intro

How it works

The API is a whole bunch of .json files that contain the data.

The API uses standard application/json format.

Why it even exists

We created it so we only had to type out data once and be able to access it everywhere else on the site, and even elsewhere!

This just saves a lot of time, and adds a little more.

File system

See the complete API directory here.

Each file and directory will be addresed later.

All API URLs will include a .json file extension

Usage

Fetch GET

Making a fetch to this API is fairly simple.

First, find the URL you want to fetch. Then create an async function.

Store the URL in a variable like such:

var response = await fetch({URL})

This is our response variable, not our result variable.

Now we will make our response variable into useable data using JavaScript's .json().

var result = await response.json()

If you want to access data inside of the result (for example the status of the result), you would do it similarly to this: result.status.

Final result:

async function api() {
//MAKE REQUEST
var response = await fetch({URL})
//CONVERT RESPONSE TO json
var result = await response.json()

//LOG THE RESULT
console.log(result)
}

api/index.json

Names Values Explanation
name STRING Contains the name for the API
id STRING Holds the type of .json file
desc STRING Holds a description of the file
urls ARRAY Contains links for the API
urls[0].codes[#].name STRING The name of the link
urls[0].codes[#].type STRING Specifys the type of link
urls[0].codes[#].url STRING Holds the actual URL for the URL
wiki STRING Contains the URL for the GitHub wiki for this API

api/projects.json

Names Values Explanation
projects[#].name STRING The name for the project
projects[#].desc STRING / NULL Holds the description for the project
projects[#].state STRING Contains a "state" to the project
projects[#].state-reason STRING / NULL Contains a explanation to the current project's state

api/urls
